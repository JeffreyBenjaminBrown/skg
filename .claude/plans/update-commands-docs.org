* Update [[./docs/COMMANDS.org][COMMANDS.org]] with undocumented commands

** What COMMANDS.org currently covers

All the main user-facing commands are documented.
I compared every interactive function in the elisp/ directory
against the entries in COMMANDS.org.

** Commands missing from COMMANDS.org

*** skg-connection-end
    Defined in [[./elisp/skg-client.el][skg-client.el]].
    Closes the TCP connection to the server.
    Belongs under a new "connection management" section,
    or under "not important but cool".

*** skg-connection-verify
    Defined in [[./elisp/skg-request-verify-connection.el][skg-request-verify-connection.el]].
    Sends a ping to the server and reports whether it responds.
    Same section as skg-connection-end.

*** skg-visit-file-diff
    Defined in [[./elisp/skg-request-file-path.el][skg-request-file-path.el]].
    Probably related to skg-diff-view.
    Needs investigation: is this user-facing or internal?

*** skg-view-id-rep-id
    Alias for 'skg-request-single-root-content-view-from-id'.
    Defined in [[./elisp/skg-commands.el][skg-commands.el]].
    Seems like a lower-level version of skg-view-id-rep.
    COMMANDS.org mentions skg-view-id-rep but not this alias.
    Maybe it should be documented as an alternative entry point,
    or maybe skg-view-id-rep supersedes it.

** Proposed changes to COMMANDS.org

Add a section "** connection management" before "** not important but cool":

#+begin_src org
  ** connection management
  *** skg-connection-end
      Closes the connection to the skg server.
      The connection will be re-established automatically
      on the next command that needs it.
  *** skg-connection-verify
      Sends a ping to the server and reports whether it responds.
      Useful for debugging connection issues.
#+end_src

Add skg-request-containerward-stats description to the undocumented section,
or move it to a real section if it's user-facing enough.

   This looks like a bug. Should it get '(interactive)' added?
