** skg wish 79009c04 "Don't search a repo's .git folder."
   Survey result: every production scanner of skg sources is a flat
   (non-recursive) read_dir filtered to *.skg, so none can descend
   into .git -- EXCEPT the org-roam importer, whose WalkDir recursed
   into everything. server/import_org_roam.rs now uses
   'filter_entry' to skip .git.
** TODO.org "the diff report should include empty top-level categories"
   server/diff_analysis/render.rs now renders every bucket heading,
   empty or not, so a reader can see that (e.g.) nothing was orphaned.
   Choices made (flagged here because they were mine): the
   "IDs duplicated across sources" section is still suppressed when
   empty (it is a warning, not a routine category), and the
   "** no affected nodes" summary line is kept.
** TODO.org "strip trailing whitespace from each line at save" (also under "s > correctness")
   'divide_into_viewNodeLineCols'
   (server/from_text/buffer_to_viewnodes/uninterpreted.rs) now maps
   str::trim_end over the buffer's lines. Titles were already trimmed
   downstream; this is what strips body lines. No test pinned trailing
   whitespace, and the whole suite stayed green.
** TODO.org "don't complain if point is before first headline on save"
   skg-request-save-buffer (elisp/skg-request-save.el) now moves point
   to the first headline instead of letting org-back-to-heading raise
   "Before first headline". The nvim client already no-ops in that
   case ('owning_headline_line' returns nil), so no parity gap.
** TODO.org "remap C-c C-o during Skg in skg buffers" (under "easy")
   C-c C-o is now bound to skg-goto in skg-content-view-mode-map and
   skg-diff-analysis-mode-map (elisp/skg-keymaps-and-aliases.el),
   shadowing org-open-at-point, which cannot follow skg links.
   skg-goto was chosen over the non-interactive skg-visit-link because
   it is the canonical follow-what's-at-point command (handles links,
   raw UUIDs, metadata, filenames, and the override-choice menu).

   only the red "Oh". Changes:

* what's imperfect about what's done
  - The drawn-overrider herald change removes the "Oh" glyph
    everywhere, including for an InactiveNode substitute (which never
    had herald machinery). Since org_to_text only emits overridesHere
    for ActiveNodes, nothing is displayed either way for inactive
    substitutes -- but if a future change emits it for them, there is
    now no fallback glyph.
  - The empty-categories diff report prints bare headings with no
    body; if you would rather see an explicit "(none)" child, that is
    a two-line change in render_report.
  - The C-c C-o comment in skg-keymaps-and-aliases.el says it shadows
    org-open-at-point; that is literally true only in org-derived
    modes (content view). In skg-diff-analysis-mode it simply adds the
    binding.
  - During implementation, two concurrent test runs (mine and a
    subagent's) collided on TypeDB and produced spurious failures
    (fork::all_tests, collateral::save). Everything passes in a clean
    serial run; this matches the known "shared resource" flake family
    already recorded in problems.org.
