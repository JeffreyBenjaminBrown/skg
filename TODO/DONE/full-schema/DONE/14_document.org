* plan/record : document the resulting model
This is the final step of [[../metaplan_2.org]], "document the
resulting model".  Per Jeff's instruction, documentation has no
useful plan/execute split, so this file is a RECORD of what was
written, not a plan written ahead of execution.  Progress is logged
in [[../progress.org]] (stage 14).

* scope (from the metaplan)
The metaplan asked for: schema.tql comments; api-and-formats.md (new
stats metadata, override-choice behavior, diff-mode refusals,
visible-subset save semantics); glossary.md (col-policy vocabulary
and override resolution terms; "PartnerCol" naming decision; save
pipeline made canonical); sharing docs; command docs for
override-choice navigation; mark vision.org and metaplan 1
historical; note 7_saving-readonly-cols.org superseded; document the
independent-children-above-members col ordering at
'complete_relevant_children' and in user docs; documentation to use
graph-role vocabulary throughout.

Most per-feature doc was already written incrementally as each
feature shipped (stage 2: the warning channel + "read-only set"
glossary entry; stage 9: source-set omission/retention/visible-subset
save semantics in api-and-formats.md; stage 10: herald-rules endpoint;
stage 11: override substitution + marker + override-choice + the
override vocabulary in glossary.md + COMMANDS.org; stage 12: diff-mode
refusals).  So this step was mostly GAP-FILLING and reconciliation.
