- C's *title* contains a link to 3d9aa9be-... (the instruction

- C's *body* yields a =target_filepath=.
R is then written to =org-exports/<target_filepath>.org=.

*** Q2. How to rewrite links to a NON-root node
(A non-root node = one that appears as content somewhere, but is
not itself an export root with its own file.)
Recommendation: *deep heading-anchor links*, using org's heading
search --
  cross-file: =[[./rel.org::*Heading][label]]=
  same-file:  =[[*Heading][label]]=
This lands the reader on the exact section (you wanted this if org
permits it; it does). Caveat: org matches heading TEXT, so if two
headlines in the target file share text it resolves to the first;
I'll warn when that happens.
Alternative: link only to the target file's top
(=[[./rel.org][label]]=) -- fully robust, but the reader lands at
the top and must scroll.

Jeff: Agreed: deep.

*** Q3. =target_filepath= is not valid TOML in your data
The instruction node says the marker body "is interpreted as
.toml", and both it and the real marker (512ea9da) write:
: target_filepath = org_export_is_partial
But that is *not* valid TOML -- a bare unquoted string like
=org_export_is_partial= is rejected by a strict TOML parser (TOML,
unlike YAML, requires string values to be quoted).
Recommendation: a *tolerant* line parser accepting the value bare
OR quoted (=foo= and ="foo"= both work), so current data exports
now and quoted-TOML still works later. I'll note this in
TODO/problems.org.
Alternative: enforce strict TOML -- means quoting the value in the
data and in the instruction node's example.

Jeff: Yes, tolerant is good. But if the path might have whitespace and isn't wrapped in quotes we should refuse with an explanatory message.

*** Q4. Source-set safety (avoid leaking private notes)

a general leak risk.
Recommendation: I will NOT edit your config; the prompt offers
whatever sets exist (today: just =all=). I'll recommend you add:
: [[source_sets]]
: name = "public"
: sources = ["public"]
and export that set.
Alternative: tell me to add that =public= set (or another) for you.

But in the event that a user doesn't have any other sources set up, it should permit them, but only offer the one source set (we already have code somewhere in elisp/ for scrolling circularly through a list) and explain that they might want to define a public-facing subset.

** Minor defaults (FYI, not blocking -- flag if you dislike)
- Root node => level-1 headline =* <title>=; nesting adds one =*=
  per content level. No =#+title:= (matches your README/
  this-is-awesome/setup style).
- A non-root node reachable under two different export files (your
  "content with two containers" case) gets ONE canonical home for
  link targets: export roots sort by target_filepath, and the
  node's home is its first appearance in that order.
- Broken links (target not exported under the chosen set) point to
  the export of 9ff04e25 (=org_export_is_partial.org=), per your
  instruction node. If that sink isn't exported under the chosen
  set, the link degrades to plain label text + a warning.
- Relative links use your =[[./path.org][label]]= house style.
