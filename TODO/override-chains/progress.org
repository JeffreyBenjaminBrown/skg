* What's done and went well
** DONE Phase A — invariants: drop "no chains", add "no cycle", dedup the walk
   - A1: 'OverrideResolution' gained a 'cycle : Vec<ID>' field
     (override_resolution.rs). On a detected cycle it now carries the
     trail from the first repeat back to it (walk order); empty
     otherwise. 'effective'/'path' semantics for substitution callers
     unchanged (input / empty on a cycle).
   - A2: renamed 'OverrideInvariantViolation::UserOwnedOverrideChain'
     -> 'UserOwnedOverrideCycle { cycle : Vec<ID> }'; formatter prints
     "user-owned override cycle: a -> b -> ... -> a".
   - A3: global 'validate_override_invariants' keeps pass 1 (monogamy);
     pass 2 now walks each user-owned node via 'resolve_override(None)'
     and reports a returned cycle. Cycles are canonicalized (rotate to
     min pid) and deduped via a local HashSet so the same cycle from
     different entry points collapses to one violation.
   - A4: 'validate_touched_override_invariants' keeps the monogamy
     (target-end) check; the two chain checks are replaced by one
     'resolve_override(None)' walk from each touched user-owned node.
   - A5: tests updated (see "order" note). All 35 override tests pass.

** DONE Phase B — tamper check accepts any honest carrier on the chain
   - B1: added 'carrier_on_user_owned_chain' to override_resolution.rs:
     true iff the carrier is ON the original's user-owned override chain
     ('resolve_override(None).path.contains(carrier)'), so a middle
     carrier (drawn when a later link's source is hidden) passes and an
     off-chain marker still fails.
   - B2: 'overridesHere_marker_errors' now uses that predicate instead
     of '== effective'. Kept the fail-closed branch (graph handle
     absent, or marker on an id-less node). Updated the
     'OverridesHere_Mismatch' message: the carrier is "not on the
     override chain of" the original.
   - B3: new on-disk chain fixture
     (tests/override_substitution/fixtures-chain/) D overrides C
     overrides N, end D in source 'other'. New sub-test
     'chain_half_visible_keeps_the_original': under 'all' the end D is

     the half-visible view keeps N in P's contains; an off-chain marker

* Unanticipated future potential costs and benefits
  - The global validator's cycle scan calls 'resolve_override' once per
    user-owned node without memoizing proven-acyclic pids (the plan
    permitted skipping memoization on the cold init/rebuild path). Cost
    is O(V · chain depth) worst case; chains are short in practice.
    Cheap to add memoization later if init time ever matters.
