#+TITLE: Vognode/Phantom split -- implementation plan
#+AUTHOR: plan for Jeff + future AI implementer

This document is the implementation plan for splitting the placeholder
("phantom") kinds out of 'Vognode' into a new sibling enum 'Phantom', and
renaming 'Vognode::Normal' to 'Vognode::Active'. The same split is mirrored on
the maybe-placed side ('MpVognode' / new 'MpPhantom' / 'MpViewnodeKind::Phantom').

Two mechanical renames land FIRST, each as its own commit before the split (see
Naming decisions and Step 0): 'should_be_phantom' -> 'should_be_diffPhantom', and
the phantom payload structs 'DiffPhantomNode'/'DeletedNode'/'UnknownNode' ->
'PhantomDiff'/'PhantomDeleted'/'PhantomUnknown'. Every code sample and checklist
item below uses the POST-rename names.

Code references are wrapped in single apostrophes (e.g. 'Vognode::Active'). All
file paths are absolute.

* Goal

Three moves, all internal to Rust; the external wire format does not change.

** (a) 'Vognode::Normal' becomes 'Vognode::Active'.
** (b) The three display-only placeholder kinds leave 'Vognode' for a NEW sibling enum 'Phantom', reached as 'ViewNodeKind::Phantom (Phantom::Diff (..))' etc. 'DiffPhantom' is renamed to 'Diff' in the move.
** (c) 'Inactive' STAYS in 'Vognode'. 'Active' and 'Inactive' are the two graph-backed kinds.

* Hard constraints

** Wire-format invariance
The external metadata wire format is the Emacs client API and MUST stay
byte-identical. Concretely, the atoms emitted in
'/home/ubuntu/host/server/org_to_text.rs' and parsed in
'/home/ubuntu/host/server/serve/parse_metadata_sexp.rs' -- '(node ...)',
'(deleted (id X)(source S))', '(unknownNode (id X))', 'inactiveNode', the
scaffold atoms -- must not change by a single byte. Only the internal Rust match
structure changes; never the emitted/accepted strings, and never the bodies or
signatures of the '*_metadata_to_string' helpers.

The internal error-label strings ('deleted:', 'unknown:', 'inactive:' in
'MpViewnode::error_label') and the validation-message text are NOT wire format,
but should also stay byte-identical to avoid changing diagnostic output.

** Jeff's formatting/naming rules
- Verbose names; mixed camel/snake so compound words read as one token.
- Every 'let' and 'let-else' gets a type signature.
- In a type signature, the name, each argument, and the return type each go on
  their own line.
- NO internal 'use' statements or '::'-qualified paths outside a file's top 'use'
  preamble. The new 'Phantom' / 'MpPhantom' names MUST be added to the relevant
  files' top preambles wherever they are pattern-matched.
- Trailing brackets sit on the last line of what they close.
- Group adjacent punctuation.
- PRESERVE existing comments (relocate rather than delete unless truly obsolete).
- Do NOT reformat untouched code.
- Never add history-narration comments ("we used to...").
- Callers go before callees within a module.

** Two collision/scope warnings
- '/home/ubuntu/host/server/types/phantom.rs' ALREADY EXISTS. It is the
  unrelated git-diff phantom-lookup module (a child of 'crate::types::git',
  exporting 'title_for_phantom' / 'phantom_axes'). It does NOT hold the new
  'Phantom' enum. Define the new 'enum Phantom' INSIDE 'viewnode.rs' -- do not
  create or overwrite a 'phantom' module.
- The struct payload types are NOT renamed. Only the enum variant and the
  containing enum change.

#+begin_src rust
impl ViewNode {
  pub fn normal_to_phantom (
    &mut self,
  ) {
    if let ViewNodeKind::Vognode (Vognode::Active (t))
      = &self . kind
      { if t . should_be_diffPhantom ()
        { // A phantom is ALWAYS indefinitive and renders no body (Jeff's
          // TODO/DONE/local-view-update/progress.org §9 TODO): the diff view presumes git literacy (magit
          // shows the real node), and a phantom must never be a node's
          // definitive instance -- if Removed it has nothing to define, and if
          // RemovedHere "edit it here, where it isn't" is dangerously
          // confusing. So drop body/edit_request when flipping a (possibly
          // Definitive) Active node to a phantom. mk_phantom_viewnode already
          // builds from an Indefinitive base, so now every phantom is
          // indefinitive by construction.
          let phantom : PhantomDiff =
            PhantomDiff::from_truenode ( t . clone () );
          self . kind = ViewNodeKind::Phantom (
            Phantom::Diff (phantom)); }}}

  pub fn title (&self) -> &str {
    match &self . kind {
      ViewNodeKind::Vognode (Vognode::Active (t)) => &t . title,
      ViewNodeKind::Phantom (Phantom::Diff (p)) => &p . title,
      ViewNodeKind::Phantom (Phantom::Deleted (d)) =>
        &d . title,
      ViewNodeKind::Qual (q) =>
        q . title (),
      ViewNodeKind::QualCol (_)
        | ViewNodeKind::PartnerCol (_)
        | ViewNodeKind::BufferRoot
        | ViewNodeKind::DeadScaffold
        | ViewNodeKind::Phantom (Phantom::Unknown (_))
        | ViewNodeKind::Vognode (Vognode::Inactive (_)) =>
        "",
    }}

  /// Reasonable for both TrueNodes and Scaffolds.
  pub fn body (&self) -> Option < &String > {
    match &self . kind {
      ViewNodeKind::Vognode (Vognode::Active (t)) => t . body (),
      ViewNodeKind::Phantom (Phantom::Diff (p)) => p . body (),
      ViewNodeKind::Phantom (Phantom::Deleted (d)) => d . body . as_ref (),
      ViewNodeKind::Phantom (Phantom::Unknown (_))
        | ViewNodeKind::Vognode (Vognode::Inactive (_))
        | ViewNodeKind::QualCol (_)
        | ViewNodeKind::Qual (_)
        | ViewNodeKind::PartnerCol (_)
        | ViewNodeKind::BufferRoot
        | ViewNodeKind::DeadScaffold => None,
    }}

  pub fn is_truenode_and_parentIs_affected (&self) -> bool {
    match &self . kind {
      ViewNodeKind::Vognode (Vognode::Active (t)) =>
        t . parentIs == ParentIs::Affected,
      _ => false,
    }}

  pub fn id_if_vognode (&self) -> Option<&ID> {
    match &self . kind {
      ViewNodeKind::Vognode (v) => Some (v . id ()),
      _ => None,
    }}

  /// The id of an Active vognode or a Diff phantom -- the two TrueNode-ish kinds,
  /// which before the Vognode/Phantom split both lived in Vognode and were
  /// reached by `Vognode::normal_or_phantom_id`. None for everything else
  /// (Inactive, Deleted/Unknown phantoms, cols, scaffolds, BufferRoot).
  pub fn active_or_diff_phantom_id (&self) -> Option<&ID> {
    match &self . kind {
      ViewNodeKind::Vognode (Vognode::Active (t)) => Some (&t . id),
      ViewNodeKind::Phantom (Phantom::Diff (p))   => Some (&p . id),
      _ => None,
    }}
}

** Cross-file silent-drop: 'root_ids_from_viewforest' (views_state.rs:191-194)
