* does show herald

These are metadata forms that currently produce visible herald text in
=heralds-minor-mode.el=.

** Scaffold metadata

- =aliasCol= -> =aliases=
- =alias= -> =alias=
- =hiddenInSubscribeeCol= -> =hiddenIn=
- =hiddenOutsideOfSubscribeeCol= -> =hiddenOut=
- =subscribeeCol= -> =subscribees=
- =idCol= -> =IDs=
- =id= -> =ID=
- =(textChanged staged)= -> =text changed : staged=
- =(textChanged unstaged)= -> =text changed : unstaged=
- =(deletedScaffold KIND)= -> =DELETED KIND=
- =(deleted ...)= -> =DELETED=
- =(unknownNode (id ID))= -> =Parent references unknown node.=

** Git/diff axes

For scaffolds:

- =(staged newM)= -> =staged:M=
- =(staged removedM)= -> =staged:-M=
- =(unstaged newM)= -> =unstaged:M=
- =(unstaged removedM)= -> =unstaged:-M=

For TrueNodes:

- =(staged newX)= -> =staged:X=
- =(staged removedX)= -> =staged:-X=
- =(staged newM)= -> =staged:M=
- =(staged removedM)= -> =staged:-M=
- =(unstaged newX)= -> =unstaged:X=
- =(unstaged removedX)= -> =unstaged:-X=
- =(unstaged newM)= -> =unstaged:M=
- =(unstaged removedM)= -> =unstaged:-M=
- =notInGit= -> =diff:not-in-git=

** Parent relation

- missing =parentIs= -> ={=
- =(parentIs affected)= -> no herald
- =(parentIs independent)= -> =⊥=
- =(birth containsParent)= -> =}=
- =(birth linksToParent)= -> =←=

=(parentIs absent)= is consumed by the herald rules but intentionally
shows no herald.

** Node state

- =indef= -> =☮=, abutted to the preceding herald
- =(editRequest delete)= -> =delete=
- =(editRequest (merge ID))= -> =merge:ID=

** Graph stats A,I,O,S should be visible.

- =(graphStats (containers N) (contents M))= -> =N{M=
- =(graphStats (containers N))= -> =N{=
- =(graphStats (contents M))= -> =M= in the same contains-count herald
- =(graphStats (linksInFromContainers N) (linksInFromLeaves M))= -> =N→M=
- =(graphStats (linksInFromContainers N))= -> =N→=
- =(graphStats (linksInFromLeaves M))= -> =M= in the same links-in herald
- =(graphStats aliasing)= -> =A=
- =(graphStats extraIDs)= -> =I=
- =(graphStats overriding)= -> =O=
- =(graphStats subscribing)= -> =S=

The server parser also accepts =containsHerald= and =linksHerald= inside
=graphStats= as output-only legacy fields, but current heralds are
computed from the raw stats above.

** View stats

- =(viewStats cycle)= -> =⟳=
- =(viewStats containsParent)= -> =}=
- =(viewStats (sourceHerald TEXT))= -> =TEXT=

** View requests

- =(viewRequests aliases)= -> =req:aliases=
- =(viewRequests containerwardView)= -> =req:containers=
- =(viewRequests sourcewardView)= -> =req:sources=
- =(viewRequests definitiveView)= -> =req:definitive=

* DONE could show herald
None of these deserve showing.

These are metadata fields the client or server parser can read, but
which are not already listed above as visible heralds.

** View position and folding

- =focused=
- =folded=
- =bodyFolded=

These are operational markers used for point and fold restoration.
The herald rules consume =focused= and =folded= silently. =bodyFolded=
is parsed and rendered by the server, and is handled by the folding
client code.

** Identity and source

- =(node (id ID) ...)=
- =(node (source SOURCE) ...)=
- =(deleted (id ID) ...)=
- =(deleted (source SOURCE) ...)=
- =(unknownNode (id ID))=

Some enclosing forms that carry these fields already have heralds, but
the individual =id= and =source= values are generally consumed silently.
The exception is scaffold =id=, whose bare scaffold form shows =ID=.

** Parent relation with no visible herald

- =(parentIs absent)=

This is intentionally heraldless because it means there is no visible
parent relationship to show.

** Explicit defaults and absent values

- omitted =parentIs=, equivalent to =Container=
- omitted =editRequest=, equivalent to no edit request
- omitted =viewRequests=, equivalent to no view requests
- omitted =graphStats= entries suppressed by the server, such as common
  container-count defaults
- omitted =viewStats= entries, equivalent to false for those view facts

These could theoretically receive heralds, but doing so would make the
display noisier by showing defaults.

** Body/title outside the metadata sexp

- headline title text
- body text below the headline

These are not metadata fields inside the =skg= sexp, but client code
obviously reads and edits them alongside metadata.
