* changes to the plan's envisioned order
** Ownership derives from the RESOLVED path, not the raw first component.
The plan sketched "first path component == owned_folder". The
integration harnesses generate configs whose sources use absolute
paths, so the shipped rule is the more truthful "absolute path
sits under DATA_ROOT/OWNED_FOLDER" (after 'make_paths_absolute'),
which subsumes the relative case. Consequence: a config file must
live in the data root for its owned/ folder to be recognized —
the integration harnesses now write their generated configs into
each test's data/ directory instead of /tmp.
** The author-folder rearrangement script was written now, not at =migration=.
The hard error would otherwise have bricked Jeff's dev server the
moment this commit landed. The script ran on data/ (all eight
sources moved under data/owned/); the accordion-level
'skg-migrate-to-accordions' remains at its planned position.

** Old Tantivy indexes and per-alias levels.
The index schema did not change, only the VALUES in alias
documents' source_field. Every real adoption path forces a full
rebuild (the author-folders script edits the config, which
invalidates incremental init; the migrate endpoint rebuilds
outright), and pre-accordion data has aliases only in home
sections, where old and new values coincide -- so no forced
rebuild was added.
** Import parse-time placeholder levels.
'nodecomplete_from_section_tree' tags lists with a placeholder
source; the caller re-tags on assignment now (import_org_roam.rs),
so levels are honest, but the parse-time placeholder remains a
wart until importers construct via the fold.

I moved ofiscal directly after public so the old named set
{public, ofiscal} survives as the prefix through ofiscal, but the
relative privacy of pers-i/pers-m/pers-p/pers-deprecated/decid/

for Jeff's current data (everything owned). Correct, but scripts
distinguishing them only matter once a foreign source exists.

* choices determining the future
** SourceSetName survives as a type.
Prefix sets are "named by their most private source", so the
wire/type vocabulary ("source-set") stays honest and the endpoint
shapes are unchanged; only 'SourceSet' (the struct) died.
** The owned/ dir name collides gracefully.
A source whose path IS the owned folder ("owned") is legal (the
fork fixtures use it); its defaulted herald label falls back to
the full name rather than an empty string.
