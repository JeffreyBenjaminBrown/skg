* DONE containerward path graphnodestat
  There's currently code to get a containerward path.
  Each can be described by three statistics:
  the path length (minimum 0),
  how many nodes are at the fork end (minimum 1),
  and whether it cycles.

  Let's add an optional field to graphnodestats
  to convey those three values,
  and for search results, compute them.
  Also add a way to request them from any node,
  similar to skg-request-containerward-view.
  (Extract the duplicate code so that both definitions
  can be in terms of the same higher-level function,
  and swallow any similar definitions into the same HLF.)

  Let's also explain to herald-minor-mode how to display them.
  The token should be a blue 'fork\length',
  where fork and length are integers,
  and the \ is instead some symbol indicative of cycling
  if there's a cycle, and 'fork' is omitted entirely if unity.
** recursive properties in TypeQL
   Does TypeQL have a good way to express these functions?
   I'll use Haskell notation
   - root :: node -> bool, true if node plays not the content role
   - fork :: node -> bool, true if it plays that role more than once
   - cycle :: node -> bool, true if it is its own contains-ancestor
   - self-contained :: node -> bool, true if it contains itself
   - end-of-containerward-path :: node -> node. For any n, if root n, self-contained n, or fork n, then end-of-containerward-path n = n. Otherwise, it is equal to distinct-end-of-containerward-path n
   - distinct-end-of-containerward-path :: node -> node. For any n, if root n or fork n or cycle m n, then distinct-end-of-containerward-path m = n. Otherwise it equals distinct-end-of-containerward-path m (container n), which we know exists because n is not a root or a fork.
