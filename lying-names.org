* how this was made
** what I requested from Claude
Please write a report, 'lying-names.org',
describing all functions in server/
whose names imperfectly capture what they do.
For each name it should say
- the name (as the headline),
- whether the name is true,
- whether it omits anything

For instance, I recently found that a function called
read_all_skg_files_from_sources
was also checking for duplicate IDs.
So I renamed it to
read_all_skg_files_from_sources_AND_check_for_dup_ids.
(I'll go on to decouple those two things,
but that's out of scope for this request.)

This often will be a judgment call.
If the name could perfectly express intent,
we would not need the code.
But it's okay if you over-report,
describing imperfections in the names that I won't care about.
Better than than under-reporting.
** what Claude did
Auto-generated by three parallel audit agents covering distinct
subdirectories of server/. Each function entry says whether the
function name is true and what (if anything) the name omits.

Coverage: every .rs under server/ except test_utils*, abandoned/,
and lib.rs.
